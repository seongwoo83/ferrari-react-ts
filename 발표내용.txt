1. 늘어나는 모바일 이용자를 위한 화면크기 고려 디자인
-> 최소 500px의 화면 사이즈에 맞춤 디자인
->> 모바일 이용자 뿐만 아니라 태블릿, 데스크탑 이용자들을 위한 미디어쿼리 역시 디자인 되어있음

2. 타입스크립트를 사용한 정적인 타입지정
-> 컴파일 과정의 오류 검증으로 빠른 유지 보수 가능
-> 각 데이터들을 type alias 를 통해 각 항목에 대한 타입들을 미리 선언해두고 그에 맞는 값만 입력하도록 지정함
-> 각 데이터들이 객체가 아니라 배열로 저장되어 있기 때문에 interface를 이용하지 못함
-> 타입 export로 똑같은 타입을 가진 배열은 동일한 타입설정


4. state와 property 전달로 각 라우터사이의 데이터를 전송함

5. useLocation 사용으로 게시글의 각 고유번호를 전달하고 다음, 이전 버튼 클릭시 해당 게시글의 고유번호를 state 전달로 가지고 옴

6. 이전 2차 프로젝트에 직접 만들어 사용했던 이미지 전환 모듈을 다시 이용하여 코드의 재사용성을 증대시킴
